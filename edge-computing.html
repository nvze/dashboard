Edge Computing

Pemrosesan data semakin bergeser dari cloud ke perangkat tepi (edge) seperti sensor IoT, kendaraan otonom, dan perangkat mobile. Ini memungkinkan analisis real-time dengan latensi rendah dan mengurangi beban bandwidth.

Konsep Inti: Daripada mengirim semua data mentah ke "pusat" (Cloud) yang jauh, kita memprosesnya langsung di "lokasi kejadian" (Edge).

1. Ilustrasi: "Otak vs. Refleks"

Bayangkan tubuh manusia sebagai sistem jaringan:

Cloud (Awan/Pusat) = Otak Anda

Otak sangat cerdas dan bisa memikirkan hal kompleks, tapi posisinya ada di kepala (jauh dari tangan/kaki).

Edge (Tepi) = Sistem Syaraf Refleks

Ini tidak secerdas otak, tapi posisinya langsung di tempat kejadian (tangan).

Skenario: Anda tidak sengaja menyentuh panci panas.

EDGE (Tangan)
Jarak Jauh (Latensi)
CLOUD (Otak)

Cara Lama (Cloud Processing)

Jari menyentuh panas → Kirim sinyal ke Otak → Otak memproses: "Ini panas, bahaya!" → Otak kirim perintah balik: "Tarik tangan!" → Tangan ditarik.

Masalah: Terlalu lama. Tangan Anda sudah keburu terbakar sebelum perintah sampai (Latensi tinggi).

Cara Baru (Edge Processing)

Jari menyentuh panas → Syaraf di tangan langsung memproses → Tangan ditarik seketika (Refleks) → Baru setelah itu melapor ke otak: "Tadi saya kena panas."

Hasil: Tindakan instan (Real-time), kerusakan minimal.

2. Membedah Istilah Teknis

Berikut adalah arti dari poin-poin dalam teks awal berdasarkan ilustrasi di atas:

A. "Analisis Real-time dengan Latensi Rendah"

Maksudnya: Tidak ada jeda waktu tunggu (delay/loading).

Contoh pada Kendaraan Otonom (Mobil Tanpa Sopir):
Jika ada anak kecil menyeberang mendadak, mobil harus mengerem dalam hitungan milidetik.

  • Jika Cloud: Mobil harus "bertanya" dulu ke server pusat: "Itu gambar apa? Apa saya harus ngerem?", maka akan terjadi tabrakan karena butuh waktu kirim data bolak-balik.
  • Jika Edge: Komputer di dalam mobil memutuskan sendiri saat itu juga untuk mengerem.

B. "Mengurangi Beban Bandwidth"

Maksudnya: Menghemat kapasitas "pipa" internet.

Ilustrasi CCTV Keamanan:

  • Cloud: CCTV mengirim video 24 jam non-stop ke pusat, padahal 90% isinya hanya jalanan kosong. Ini memacetkan jaringan internet (Boros Bandwidth).
  • Edge: CCTV pintar hanya mengirim data jika mendeteksi kecelakaan atau pelanggaran. Sisa waktunya dia diam. Ini membuat internet jauh lebih lancar.

3. Kenapa Pergeseran Ini Terjadi?

Dulu, perangkat kecil (HP, sensor) tidak punya "otak" (processor) yang kuat, jadi semua harus dikirim ke pusat.

Sekarang, perangkat kecil (Sensor IoT, HP) sudah punya processor yang sangat canggih dan kuat. Mereka mampu berpikir sendiri tanpa harus selalu bergantung pada pusat. Cloud tetap digunakan, tapi hanya untuk analisis jangka panjang atau penyimpanan data arsip, bukan untuk keputusan detik-demi-detik.
